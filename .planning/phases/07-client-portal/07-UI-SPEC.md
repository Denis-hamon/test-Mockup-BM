## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, message bubble internal icon spacing |
| sm | 8px | Chat bubble padding inline, compact element gaps, timestamp spacing |
| md | 16px | Default element spacing, card padding inline, message list gap |
| lg | 24px | Section padding, card padding block, sidebar section spacing |
| xl | 32px | Layout gaps between major sections (dashboard summary cards) |
| 2xl | 48px | Page section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions: 44px minimum touch target for mobile sidebar nav items and send button.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Message text, case descriptions, form fields, document names |
| Label | 14px | 600 (semibold) | 1.4 | Sidebar nav items, section titles, field labels, timestamps |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles ("Mes dossiers", "Messages"), case name in header |
| Display | 28px | 600 (semibold) | 1.2 | Not used in this phase (reserved for landing) |

Note: Only 2 weights used -- 400 (regular) for body/content, 600 (semibold) for labels/headings/emphasis. Consistent with Phase 6.

## Layout Contract

### Client Dashboard (/portail or /portail/accueil)

```
+----------------------------------------------------------+
| Header (shared app layout -- border-b, bg-card)          |
| LegalConnect    [Connexion securisee]    email  [client]  |
+----------+-----------------------------------------------+
| Sidebar  | Page title "Bienvenue, [Prenom]"              |
| ------   |                                                |
| Accueil  | +--card---+ +--card---+ +--card---+            |
| Dossiers | | Dossiers | | Messages | | Prochain |         |
| Messages | | 2 actifs | | 3 non   | | RDV      |         |
| RDV      | |          | | lus     | | 15 avr   |         |
| Documents| +----------+ +---------+ +----------+         |
| Params   |                                                |
|          | Activite recente:                               |
|          | [Timeline of last 5 events across all cases]   |
+----------+-----------------------------------------------+
```

- Sidebar: 240px fixed width on desktop, Sheet on mobile (< 768px)
- Sidebar: shared structure with lawyer dashboard, content conditional on role
- Summary cards: 3-column grid on desktop, single column on mobile
- Activity timeline: max 5 recent events, "Voir tout" link to case

### Case List (/portail/dossiers)

```
+----------+-----------------------------------------------+
| Sidebar  | Page title "Mes dossiers"                      |
|          |                                                |
|          | +--card---------------------------------+       |
|          | | [Status badge] Droit du travail       |       |
|          | | Soumis le 15 mars 2026                |       |
|          | | Dernier message: il y a 2h  [3 non lus]|      |
|          | +--------------------------------------+       |
|          | +--card---------------------------------+       |
|          | | [Status badge] Droit de la famille    |       |
|          | | Soumis le 10 mars 2026                |       |
|          | | Statut: Termine                       |       |
|          | +--------------------------------------+       |
+----------+-----------------------------------------------+
```

- Cases displayed as cards (not table) -- client has fewer cases, cards more appropriate
- Card shows: status badge, problem type, submission date, last message or status
- Unread indicator: blue dot + count on cards with unread messages
- Sort: most recent activity first (no manual sorting for client)

### Case Detail (/portail/dossiers/[id])

```
+----------+-----------------------------------------------+
| Sidebar  | < Retour a mes dossiers                        |
|          | [Client case title]  [Status badge]            |
|          | CaseStatusTracker: o---o---o                    |
|          |                   Nouveau En cours Termine      |
|          +-----------------------------------------------+
|          | [Suivi] [Documents] [Messages]                  |
|          +-----------------------------------------------+
|          | Tab content:                                    |
|          |                                                |
|          | Suivi: Simplified timeline of events            |
|          | Documents: Grid of uploaded files + download     |
|          | Messages: Inline ChatContainer                  |
+----------+-----------------------------------------------+
```

- CaseStatusTracker: horizontal progress with 3 steps, current step filled with primary color
- Tabs: 3 tabs only (client sees Suivi, Documents, Messages -- NOT Synthese/Echanges IA)
- Messages tab embeds the full ChatContainer inline (no separate page)
- Documents tab reuses file-preview component with download button

### Messages (/portail/messages)

```
+----------+--------------------+---------------------------+
| Sidebar  | Conversations      | Chat Area                 |
|          | +-conv-----------+ | [Droit du travail - Me X] |
|          | | Me Dupont       | | [Chiffre bout en bout]   |
|          | | Droit travail   | |                           |
|          | | Dernier msg...  | | [bubble] Bonjour...       |
|          | | il y a 2h  [3] | |       [bubble] Merci... |
|          | +-conv-----------+ |                           |
|          | +-conv-----------+ | [TypingIndicator...]      |
|          | | Me Martin       | |                           |
|          | | Droit famille   | | +------------------------+|
|          | | Aucun message   | | | [clip] Message... [>] ||
|          | +-conv-----------+ | | [lock] Chiffre E2E     ||
|          |                    | +------------------------+|
+----------+--------------------+---------------------------+
```

- Split view: conversation list (280px) + chat area (flex-1)
- Mobile: full-screen conversation list, tap opens full-screen chat with back arrow
- Conversation list items: lawyer name, case type, last message preview (truncated 60 chars), relative time, unread count badge
- Chat area header: lawyer name + case type + encryption badge
- Message list: scroll-area with infinite scroll (load 50 messages, paginate on scroll up)
- ChatInput: single line expanding to max 4 lines, send button (primary), attach button (ghost), padlock icon left of input
- Sent messages: right-aligned, primary background, primary-foreground text
- Received messages: left-aligned, muted background, foreground text
- Timestamps: muted-foreground, 12px, below bubble, grouped by date separators
- Read receipts: "Lu" text in muted-foreground, 12px, below sent message timestamp (only when lawyer has enabled)
- File attachments in messages: compact card below bubble text with file icon + name + size + download link

+----------+-----------------------------------------------+

- Confirmed visio appointments show a clickable link
- Confirmed presentiel appointments show cabinet address
- "En attente" appointments show "En attente de confirmation de votre avocat"
- Cancel button on "En attente" appointments only, with alert-dialog confirmation

## Interaction Contract

### Client Dashboard

| Interaction | Behavior |
|-------------|----------|
| Summary card click | Navigate to corresponding section (dossiers, messages, rendez-vous) |
| Activity timeline item click | Navigate to the relevant case detail |

### Case List

| Interaction | Behavior |
|-------------|----------|
| Case card click | Navigate to case detail page |
| Unread badge visible | Blue dot + count, disappears after opening messages tab |

### Chat / Messaging

| Interaction | Behavior |
|-------------|----------|
| Send message | Enter key or click send button. Message appears instantly (optimistic). Encrypted client-side before transmission. |
| Attach file | Click paperclip icon, file picker opens. Reuses file-dropzone pipeline. File encrypted before upload. Preview shown inline in chat. |
| Scroll up | Load 50 older messages (infinite scroll with skeleton loader at top) |
| Typing indicator | Debounced 2s. Shows "[Name] ecrit..." with animated dots. Disappears after 5s of no input. |
| Read receipt | "Lu" appears under sent message when other party has viewed it. Only when lawyer has enabled read receipts. |
| Conversation switch | Click conversation in list. Chat area updates. Mobile: navigate to full-screen chat. |
| Real-time message | New message appears at bottom with subtle fade-in animation. Auto-scroll to bottom if already at bottom. If scrolled up, show "Nouveau message" floating button. |
| Encryption badge | Static display, not interactive. Tooltip on hover: "Vos messages sont chiffres de bout en bout. Personne d'autre ne peut les lire." |

### Appointments

| Interaction | Behavior |
|-------------|----------|
| "Demander un rendez-vous" button | Opens Dialog with AppointmentRequestForm |
| Submit appointment form | Validate with Zod, submit server action, close dialog, toast confirmation, card appears in list as "En attente" |
| Cancel appointment | Alert-dialog confirmation, then server action to cancel, card updates to removed state with toast |
| Visio link click | Opens link in new tab |

### Case Detail

| Interaction | Behavior |
|-------------|----------|
| Tab switch | Client-side tab change, no page reload. URL hash updated (#suivi, #documents, #messages) |
| Download document | Direct download via presigned S3 URL |
| Back link | Navigate to /portail/dossiers |
| Status tracker | Read-only visual progress indicator, not interactive |

### Settings

| Interaction | Behavior |
|-------------|----------|
| Notification toggle | Toggle, no immediate save |
| "Afficher" recovery phrase | Dialog with BIP39 words after password re-entry |
| "Exporter" data | Triggers RGPD data export, download JSON file |
| Save button | Save all settings, toast confirmation, disabled while saving |
