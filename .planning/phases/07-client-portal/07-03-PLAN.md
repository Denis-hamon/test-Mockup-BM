    **6. Create `apps/web/src/components/chat/chat-container.tsx`** (per D-01, D-04, UI-SPEC):

    This is the main chat component. Client component ("use client").

    **Props:** `conversationId: string`, `currentUserId: string`, `otherUserName: string`, `isInitiator: boolean`, `otherUserPublicKey: string` (base64).

    **State management:**
    - `messages: DecryptedMessage[]` — decrypted messages in memory
    - `conversationKeys: ConversationKeys | null` — cached derived keys
    - `isTyping: boolean` — other party typing state
    - `hasMore: boolean` — pagination flag

    **Key derivation on mount:**
    1. Read user's private key from IndexedDB (existing pattern from Phase 1)
    2. Determine crypto_kx role: if `isInitiator` -> `deriveClientKeys()`, else `deriveServerKeys()` (per pitfall 1: role based on `conversation.initiatorId === currentUserId`)
    3. Cache keys in state (per pitfall 6: avoid re-derivation)

    **Message loading:**
    1. Call `getMessages(conversationId)` on mount
    2. Decrypt each message using directional keys:
       - `senderId === currentUserId` -> decrypt with `conversationKeys.tx` (I encrypted with my tx)
       - `senderId !== currentUserId` -> decrypt with `conversationKeys.rx` (they encrypted with their tx, which equals my rx via crypto_kx symmetry)

    **SSE integration:**
    - `useSSEMessages` hook at mount
    - On `new_message` event for this conversationId: decrypt and append to messages list. If `clientMessageId` matches an optimistic message, replace it.
    - On `typing` event: set isTyping true, auto-clear after 5s
    - On `read_receipt` event: update message's readAt in state

    **Send flow:**
    1. Generate `clientMessageId = crypto.randomUUID()`
    2. Encrypt plaintext with `conversationKeys.tx` using `crypto_secretbox_easy` + random nonce
    3. Optimistic insert into messages (opacity-50 state)
    4. Call `sendMessage(conversationId, base64Ciphertext, base64Nonce, clientMessageId, attachmentId?)`
    5. On success: SSE confirms, replace optimistic
    6. On error: show red border + retry

    **Mark as read:**
    - Call `markAsRead(conversationId)` when chat is visible/focused

    **Scroll behavior:**
    - Use ScrollArea component. Auto-scroll to bottom on new messages if already at bottom.
    - "Nouveau message" floating button if scrolled up and new message arrives
    - Scroll up to top triggers `getMessages(conversationId, cursor)` for pagination

    **Layout:**
    - Header: other party name + case type + EncryptionBadge variant="header"
    - Body: ScrollArea with ChatBubble list, TypingIndicator at bottom
    - Footer: ChatInput
