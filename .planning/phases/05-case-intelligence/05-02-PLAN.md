**`triggerCaseIntelligence(submissionId: string): Promise<{ success: boolean; error?: string }>`**

Per D-02: generation automatique des la soumission.

1. Determine version number: query max version from caseSummaries where submissionId matches. If none, version = 1. Otherwise, version = max + 1.
2. If version > 1 (regeneration per D-07): update all existing rows for this submissionId in caseSummaries, caseTimelines, qualificationScores to set `isLatest = 0`.
3. Insert pending rows in all three tables:
   - `caseSummaries`: { submissionId, version, isLatest: 1, status: "pending" }
   - `caseTimelines`: { submissionId, version, isLatest: 1, status: "pending" }
   - `qualificationScores`: { submissionId, version, isLatest: 1, status: "pending" }
4. Enqueue BullMQ job: `caseIntelligenceQueue.add("generate-case-intelligence", { submissionId, version }, { attempts: 2, backoff: { type: "exponential", delay: 5000 } })`.
   Note: only 2 attempts (not 3 like extraction) because LLM calls are expensive.
5. Return `{ success: true }`.
6. Wrap in try/catch, return `{ success: false, error: message }` on failure.
