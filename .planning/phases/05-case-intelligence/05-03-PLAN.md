    **C. Timeline AI Prompt — create apps/web/src/lib/ai/prompts/timeline.ts:**

    Follow exact pattern of case-summary.ts. French language. UPL guardrails.

    TIMELINE_SYSTEM_PROMPT must instruct the AI to:
    - Extract chronological events from the client's narrative description
    - For each event: identify date (or approximate date with '~' prefix per D-04), description, source ("recit")
    - Mark low-confidence dates with confidence: "low" (dates inferred or approximate, displayed with '~' per D-04)
    - Events without any date go to undatedEvents array (per D-04: section "Non dates")
    - Sort events chronologically
    - UPL guardrails: factual extraction only, no legal advice
    - Output format: JSON matching timelineOutputSchema

    buildTimelineUserPrompt(params): same inputs as buildCaseSummaryUserPrompt plus documentSummaries (document filenames and types for source tracking).

    **D. Qualification Score AI Prompt — create apps/web/src/lib/ai/prompts/qualification-score.ts:**

    QUALIFICATION_SCORE_SYSTEM_PROMPT must instruct the AI to:
    - Evaluate the case on 3 criteria (per D-06):
      - Urgence (40%): deadlines, urgency keywords (tres_urgent, urgent), imminent risks, time-sensitive situations
      - Completude (35%): % of form fields filled (pass field count + filled count), number of documents uploaded, detail level of description
      - Complexite (25%): number of opposing parties, legal domain complexity, multiple legal areas involved, international dimension
    - Each sub-score: 0-100 integer
    - Overall score = urgencyScore * 0.40 + completenessScore * 0.35 + complexityScore * 0.25 (rounded)
    - Provide French rationale explaining the assessment
    - UPL guardrails: factual assessment only
    - Output format: JSON matching qualificationScoreOutputSchema

    buildQualificationScoreUserPrompt(params): include problemType, description, urgency, opposingParty, fullName, documentCount, filledFieldsCount, totalFieldsCount.

    **E. Generate Timeline — create apps/web/src/lib/ai/generate-timeline.ts:**

    Follow exact same pattern as generate-case-summary.ts:
    1. Load submission + documents from DB

    7. Insert into caseTimelines table (events and undatedEvents as JSON.stringify)
    8. Return { success: true, timeline } or { success: false, error }

    **F. Generate Qualification Score — create apps/web/src/lib/ai/generate-qualification-score.ts:**

    2. Check if score already exists
    3. Get AI provider
    4. If isMock → use getMockQualificationScore(problemType) from mock-provider
