# Plan 05-01: AI Case Summary Generation

## Objective

Build the AI-powered case summary generation system that takes intake submission data and uploaded documents, processes them through the LLM layer, and produces a structured "fiche synthetique" (case summary) stored in the database and retrievable via server actions.

## Context

Phase 5 builds case intelligence on top of the existing intake form (Phase 2) and AI engine (Phase 3-4). Since Phases 3-4 have not been built yet, this plan will create the necessary AI infrastructure inline: a lightweight LLM provider abstraction using Vercel AI SDK, and the case summary generation pipeline.

The intake submission schema already exists (`intakeSubmissions` table in `apps/web/src/lib/db/schema/intake.ts`) with fields for problem type, description, urgency, opposing party, desired outcome, and uploaded documents.

## Tasks

### Task 1: Case summary database schema and Zod validation
type="auto"

**Do:**
1. Add a `caseSummaries` table to `apps/web/src/lib/db/schema/case-intelligence.ts`:
   - `id` (text, PK, randomUUID)
   - `submissionId` (text, FK to intake_submissions.id, unique)
   - `summary` (text, the generated prose summary)
   - `keyFacts` (text, JSON-encoded array of key facts extracted)
   - `legalDomain` (text, refined legal domain classification)
   - `parties` (text, JSON-encoded array of identified parties with roles)
   - `amounts` (text, JSON-encoded array of monetary amounts mentioned)
   - `urgencyAssessment` (text, AI assessment of urgency: low/medium/high/critical)
   - `strengthIndicators` (text, JSON-encoded AI assessment of case indicators)
   - `aiModel` (text, which model produced the summary)
   - `aiTokensUsed` (integer, token count for cost tracking)
   - `generatedAt` (timestamp)
   - `createdAt`, `updatedAt` (timestamps)
2. Add Drizzle relations (caseSummary belongs to intakeSubmission, intakeSubmission has one caseSummary)
3. Export from schema index
4. Add Zod schemas in `packages/shared/src/schemas/case-intelligence.ts`:
   - `caseSummarySchema` for the full summary object
   - `generateSummaryRequestSchema` for triggering generation
5. Export from shared index
6. Create migration file

**Done when:**
- Schema file exists with proper types and relations
- Zod schemas validate correctly
- Schema index exports the new table

### Task 2: AI provider service and case summary generation
type="auto"

**Do:**
1. Create `apps/web/src/lib/ai/provider.ts`:
   - Export a `getAIModel()` function that returns the configured AI model
   - Use `@ai-sdk/anthropic` as primary with `ANTHROPIC_API_KEY` env var
   - Fallback logic: if no API key, return a mock provider for development
   - Export model name for tracking
2. Create `apps/web/src/lib/ai/prompts/case-summary.ts`:
   - System prompt for case summary generation in French
   - Structured output instructions (JSON schema for the summary fields)
   - UPL guardrails: never provide legal advice, only factual analysis
   - Empathetic framing guidelines
3. Create `apps/web/src/lib/ai/generate-case-summary.ts`:
   - `generateCaseSummary(submissionId: string)` function
   - Loads the intake submission + documents from DB
   - Builds the prompt with intake data (problem type, description, urgency, parties, desired outcome)
   - Calls the AI provider with structured output
   - Parses and validates the response against Zod schema
   - Stores the result in `caseSummaries` table
   - Returns the generated summary
   - Includes error handling (AI failures, validation failures)
4. Create a dev mock provider in `apps/web/src/lib/ai/mock-provider.ts`:
   - Returns realistic French case summary data
   - Used when no API key is configured (development mode)

**Done when:**
- AI provider abstraction exists with mock fallback
- Case summary generation function compiles
- Prompts include UPL guardrails and empathetic framing
- Mock provider returns valid structured data

### Task 3: Server action and API integration
type="auto"

**Do:**
1. Create `apps/web/src/server/actions/case-intelligence.actions.ts`:
   - `generateCaseSummaryAction(submissionId: string)` - triggers AI summary generation
   - `getCaseSummaryAction(submissionId: string)` - retrieves existing summary
   - Auth check: only the submission owner or assigned lawyer can access
   - Rate limiting: max 3 generation attempts per submission
2. Update `apps/web/src/server/actions/intake.actions.ts`:
   - After successful submission, auto-trigger case summary generation (fire-and-forget)
3. Create `packages/shared/src/schemas/case-intelligence.ts` exports if not already done in Task 1

**Done when:**
- Server actions exist and handle auth/rate-limiting
- Intake submission triggers summary generation
- Summary can be retrieved by submissionId
