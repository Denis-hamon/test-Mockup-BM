## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Table cell padding, compact element gaps |
| md | 16px | Default element spacing, card padding inline |
| lg | 24px | Section padding, card padding block |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page section breaks (e.g. between case list and filters header) |
| 3xl | 64px | Not used in this phase |

Exceptions: 44px minimum touch target for mobile status dropdown and filter chips.

## Layout Contract

### Case List Page (/dashboard or /dossiers)

```
+----------------------------------------------------------+
| Header (existing app layout — border-b, bg-card)         |
+----------------------------------------------------------+
| Page title "Dossiers"            [Search] [Filters btn]  |
+----------------------------------------------------------+
| Filter bar (when expanded):                               |
| [Statut v] [Specialite v] [Score v] [Date range v] [x]  |
+----------------------------------------------------------+
| DataTable                                                 |
| Client | Type | Score | Statut | Date     | Actions      |
| -------|------|-------|--------|----------|--------       |
| row    | row  | badge | badge  | relative | [Ouvrir]     |
| ...                                                       |
+----------------------------------------------------------+
| Pagination: < 1 2 3 ... 12 >                             |
+----------------------------------------------------------+
```

- Max width: max-w-7xl (inherited from app layout)
- Table: full width within container
- Column widths: Client 30%, Type 20%, Score 12%, Statut 12%, Date 14%, Actions 12%
- Row height: 48px minimum
- Pagination: server-side, 20 items per page
- Mobile: stack as cards, hide Type and Date columns below md breakpoint

### Case Detail Page (/dossiers/[id])

```
+----------------------------------------------------------+
| Header (existing app layout)                              |
+----------------------------------------------------------+
| < Retour aux dossiers                                     |
| CaseDetailHeader:                                         |
| [Client Name]  [Type badge]  [Score badge]  [Status v]   |
| Soumis le 15 mars 2026                                    |
+----------------------------------------------------------+
| [Synthese] [Documents] [Timeline] [Echanges IA]          |
+----------------------------------------------------------+
| Tab content area (min-h-[400px])                          |
|                                                           |
| Synthese tab: AI-generated case summary in card           |
| Documents tab: Grid of uploaded documents with previews   |
| Timeline tab: Vertical chronological event list           |
| Echanges IA tab: AI follow-up conversation history        |
+----------------------------------------------------------+
| Internal Notes section (always visible below tabs):       |
| [Note editor] [Enregistrer]                               |
+----------------------------------------------------------+
```

- Back link: text button with ChevronLeft icon, muted-foreground
- Header: sticky below app header on scroll (top-[49px])
- Tabs: full width, border-b underline style (not pills)
- Tab content: min-height 400px to prevent layout shift
- Notes section: separated by a border-t, always visible regardless of active tab

## Interaction Contract

### Case List

| Interaction | Behavior |
|-------------|----------|
| Row click | Navigate to case detail page |
| Column header click | Toggle sort asc/desc on that column |
| Search input | Debounced (300ms) server-side search by client name |
| Filter change | Immediate server-side refetch, URL params updated |
| Mobile filters | Open Sheet from right edge |
| Pagination | Server-side, URL param `page=N` |

### Case Detail

| Interaction | Behavior |
|-------------|----------|
| Tab switch | Client-side tab change, no page reload. URL hash updated (#synthese, #documents, #timeline, #echanges) |
| Status dropdown | Immediate save on selection, toast confirmation |
| Status to Archive | Alert dialog confirmation before save |
| Regenerate AI | Button in Synthese tab header, loading spinner replaces content during regen, toast on completion |
| Download document | Direct download via presigned S3 URL |
| Internal notes save | Explicit save button, debounced auto-save after 5s of inactivity, toast on save |
| Back link | Navigate to /dossiers preserving previous filter/page state |

### Settings

| Interaction | Behavior |
|-------------|----------|
| Specialty checkbox | Toggle, no immediate save |
| Notification switch | Toggle, no immediate save |
| Save button | Save all settings at once, toast confirmation, disabled state while saving |

| Toasts | sonner uses role=status with polite aria-live |
