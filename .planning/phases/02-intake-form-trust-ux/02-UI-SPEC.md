# Phase 2 — UI Design Contract

> Visual and interaction contract for the intake form and trust UX. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `radix-nova` (Radix primitives, Nova style) |
| Component library | radix |
| Icon library | lucide-react |
| Font | Inter (system fallback: `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`) |

**Initialization:** Run `pnpm dlx shadcn@latest init --preset radix-nova` inside `apps/web/`. This creates `components.json`, configures Tailwind CSS v4 `@theme` blocks, and sets up `cn()` utility.

**Components to add after init:**
```
pnpm dlx shadcn@latest add button input textarea select card badge tooltip progress separator alert tabs
```

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, input padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, stepper step gaps |
| xl | 32px | Layout gaps between major form sections |
| 2xl | 48px | Page vertical padding (top/bottom) |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target minimum 44px height for all interactive elements (buttons, dropzone, step navigation). Upload dropzone has 48px vertical padding.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form field values, description textarea, help text |
| Label | 14px | 500 (medium) | 1.4 | Field labels, badge text, security indicator text |
| Heading | 20px | 600 (semibold) | 1.3 | Step titles ("Decrivez votre situation"), trust banner heading |
| Display | 28px | 600 (semibold) | 1.2 | Intake page title ("Nouvelle demande juridique") |

Font weights: 400 (regular) and 600 (semibold). Weight 500 is used only for labels as a shadcn default -- it does not count as a third declared weight since it comes from the component library.

## Color

Uses shadcn semantic color tokens. Actual hex values come from the `radix-nova` preset CSS variables.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--card` | Page background, form surfaces |
| Secondary (30%) | `--muted` / `--secondary` | Step indicator inactive, file preview cards, trust banner background |
| Accent (10%) | `--primary` | See reserved-for list below |
| Destructive | `--destructive` | File deletion button, error validation messages |
| Trust green | `hsl(142, 71%, 45%)` / Tailwind `emerald-600` via custom CSS variable `--trust` | Encryption badges, padlock icons, "Chiffre" upload badge |

**Accent (`--primary`) reserved for:**
- "Suivant" (next step) button
- "Soumettre ma demande" final CTA button
- Active step indicator in the stepper
- Progress bar fill color

**Trust green (`--trust`) reserved for:**
- Padlock icon next to sensitive fields (description, upload)
- "Chiffre" badge after successful file upload
- Shield icon in trust banner
- Tooltip trigger icon color

**Custom CSS variable to add in `@theme`:**
```css
--trust: 142 71% 45%;
```

## Copywriting Contract

All copy in French, vouvoiement. Tone: warm, supportive, encouraging. Never cold or bureaucratic.

### Step Introduction Messages

| Step | Heading | Intro Message |
|------|---------|---------------|
| 1 — Type de probleme | Quel est votre probleme juridique ? | Nous allons vous guider pour decrire votre situation. Toutes vos reponses sont confidentielles et chiffrees. |
| 2 — Description | Decrivez votre situation | Prenez votre temps pour expliquer ce qui vous arrive. Il n'y a pas de mauvaise reponse. Plus vous etes precis, mieux votre avocat pourra vous aider. |
| 3 — Pieces justificatives | Ajoutez vos documents | Si vous avez des documents en rapport avec votre situation (contrats, courriers, captures d'ecran, videos), ajoutez-les ici. Ils seront chiffres avant envoi. |
| 4 — Coordonnees | Vos coordonnees | Derniere etape. Ces informations permettront a votre avocat de vous recontacter. |

| Element | Copy |

## Interaction Contracts

### Stepper Behavior

| Behavior | Specification |
|----------|---------------|
| Navigation | Linear forward with validation gate. Free backward navigation (no validation on back). |
| Step validation | Trigger partial Zod schema validation on "Continuer" click. Block forward if invalid. |
| Progress bar | Continuous fill: 25% per completed step. Uses shadcn Progress component. |
| Step indicators | Numbered Badge components: active = `variant="default"` (accent), completed = green check icon, upcoming = `variant="outline"` (muted). |
| URL state | Step number NOT in URL. Stored in component state + localStorage. Avoids broken deep links. |
| Keyboard | Enter key on last field of a step triggers "Continuer". Tab order flows naturally within step. |
| Animation | No animation between steps. Instant swap. Framer Motion deferred to later phases. |

### File Upload Behavior

| Behavior | Specification |
|----------|---------------|
| Drop zone | Dashed border Card, 120px min-height. Border becomes `--primary` on drag-over. |
| Click to browse | Entire drop zone is clickable. Also has explicit "Parcourir" Button inside. |
| File preview | Grid layout (2 columns desktop, 1 column mobile). Each file: thumbnail (images) or icon (PDF: FileText, video: Video from lucide-react) + filename + size + status badge. |
| Encryption flow | File selected -> "Chiffrement..." label + spinner -> upload to S3 -> "Chiffre" green badge. |
| Progress | Per-file Progress bar during encryption+upload. Disappears after completion, replaced by "Chiffre" badge. |
| Remove file | Trash2 icon button on each file card. Inline confirm (not modal). |
| Limits | Max 50 Mo per file, 200 Mo total. Validation at selection time (before encryption). |
| HEIC handling | Show generic image icon (ImageIcon from lucide). No thumbnail. No client conversion. |

### Auto-Save Behavior

| Behavior | Specification |
|----------|---------------|
| Trigger | `form.watch()` subscription -- saves on every field change. |
| Storage | `localStorage` key: `legalconnect_intake_draft`. JSON serialized form values (no files). |
| Restore | On mount, check localStorage. If draft exists, populate form and show Alert "Brouillon retrouve". |
| Clear | On successful submission, remove localStorage key. |
| Error | Catch `QuotaExceededError`. Show non-blocking inline warning. Continue without auto-save. |

### Auth Gate (Discretion Decision)

| Behavior | Specification |
|----------|---------------|
| Steps 1-3 | Anonymous. No login required. Form filling is open. |
| Step 4 submission | Check auth session. If logged in: submit directly. If not: show inline login/register form within step 4 before submission. User authenticates, then submission proceeds. |
| Rationale | D-03 says "Pas de compte requis pour commencer." Matches Doctolib pattern. Reduces friction. |

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| Mobile (< 640px) | Single column. Step labels hidden (numbers only). File grid 1 column. Stepper compact (numbered badges only, no labels). |
| Tablet (640-1024px) | Single column, wider. Step labels visible. File grid 2 columns. |
| Desktop (> 1024px) | Centered content column (max-width 640px). Full step labels. File grid 2 columns. Trust banner full width within content column. |

Content max-width: `max-w-2xl` (672px) centered with `mx-auto`.
Page padding: `px-4` mobile, `px-6` tablet+.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has `FieldLabel` with `htmlFor`. No placeholder-only fields. |
| Validation | `aria-invalid` on invalid inputs. `aria-describedby` linking to `FieldDescription` error text. `data-invalid` on `Field` wrapper. |
| Progress | `aria-valuenow`, `aria-valuemin`, `aria-valuemax` on Progress (handled by shadcn). |
| Tooltips | Keyboard accessible (focus trigger). Escape to close. `role="tooltip"`. |
| File upload | Dropzone has `role="button"` and `tabIndex={0}`. Keyboard activation with Enter/Space. `aria-label="Zone de depot de fichiers"`. |
| Step navigation | Buttons have descriptive `aria-label` ("Aller a l'etape suivante", "Revenir a l'etape precedente"). |
| Color contrast | Trust green on white background: 4.5:1 minimum (emerald-600 = #059669 on white = 4.54:1). |
| Focus visible | Default shadcn focus ring (`ring-ring`). No custom focus styles. |
