**Upload Server Action** (`apps/web/src/server/actions/upload.actions.ts`):
- "use server" directive
- Import `PutObjectCommand` from `@aws-sdk/client-s3`, `s3Client` from `@/lib/s3`
- Import `crypto` from `node:crypto`

```typescript
export async function uploadEncryptedFile(formData: FormData): Promise<{ s3Key: string } | { error: string }> {
  try {
    const encryptedBlob = formData.get("file") as Blob;
    const fileName = formData.get("fileName") as string;
    const ssecKeyBase64 = formData.get("ssecKey") as string;

    if (!encryptedBlob || !fileName || !ssecKeyBase64) {
      return { error: "missing_fields" };
    }

    const s3Key = `intake/${crypto.randomUUID()}/${fileName}`;
    const ssecKey = Buffer.from(ssecKeyBase64, "base64");
    const ssecKeyMd5 = crypto.createHash("md5").update(ssecKey).digest("base64");

    await s3Client.send(
      new PutObjectCommand({
        Bucket: process.env.OVH_S3_BUCKET!,
        Key: s3Key,
        Body: Buffer.from(await encryptedBlob.arrayBuffer()),
        SSECustomerAlgorithm: "AES256",
        SSECustomerKey: ssecKeyBase64,
        SSECustomerKeyMD5: ssecKeyMd5,
      })
    );

    return { s3Key };
  } catch (error) {
    console.error("Upload failed:", error);
    return { error: "upload_failed" };
  }
}
```

**File Encryption Hook** (`apps/web/src/hooks/use-file-encryption.ts`):
- "use client" directive
- Import `encrypt` from `@legalconnect/crypto`, `sodium` from `libsodium-wrappers-sumo`
- Import `useState`, `useCallback` from `react`

Types:
```typescript
interface UploadedFile {
  id: string;
  name: string;
  size: number;
  type: string;
  encryptedKey: string; // base64 sealed box
  s3Key: string;
  nonce: string; // base64
  status: "encrypting" | "uploading" | "done" | "error";
  progress: number; // 0-100
  thumbnailUrl?: string; // for image preview via createObjectURL on original file
}
```

`useFileEncryption()` hook returns `{ files, encryptAndUpload, removeFile }`:

`files` state: `UploadedFile[]`

`encryptAndUpload(file: File)`:
1. Generate UUID for file id: `crypto.randomUUID()`
2. Add file to `files` state with status "encrypting", progress 0
3. If file is an image (not HEIC), create thumbnail URL via `URL.createObjectURL(file)` and store in `thumbnailUrl`
4. Read file as `Uint8Array` via `new Uint8Array(await file.arrayBuffer())`
5. Generate random per-file symmetric key: `sodium.randombytes_buf(sodium.crypto_secretbox_KEYBYTES)` (32 bytes)
6. Encrypt file content: `const { ciphertext, nonce } = await encrypt(fileBuffer, fileKey)`
7. For anonymous users (no permanent keypair yet), use a temporary ephemeral keypair:
   - Generate ephemeral keypair: `sodium.crypto_box_keypair()`
   - Seal the file key: `sodium.crypto_box_seal(fileKey, ephemeralKeypair.publicKey)`
   - Store ephemeral private key in sessionStorage for later re-encryption at registration
   - This solves the open question from RESEARCH.md about encrypting files for anonymous users
8. Derive SSE-C key from file key via HKDF (per RESEARCH.md Pitfall 3 recommendation):
   - Use `sodium.crypto_generichash(32, fileKey)` to derive a 32-byte SSE-C key deterministically from the file key
   - Convert to base64 for S3 header
9. Update status to "uploading", progress 50
10. Create FormData with encrypted blob, fileName, ssecKey
11. Call `uploadEncryptedFile(formData)`
12. If success: update file status to "done", progress 100, store s3Key
13. If error: update file status to "error"

`removeFile(fileId: string)`:
- Filter file out of `files` state
- If file had `thumbnailUrl`, call `URL.revokeObjectURL(thumbnailUrl)`
