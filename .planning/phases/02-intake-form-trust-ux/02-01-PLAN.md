**Step 5: Create Zod intake schemas (per D-02, 4-step structure)**
Create `packages/shared/src/schemas/intake.ts` with these exact exports:
- `stepProblemTypeSchema`: z.object with `problemType` (enum: "famille", "travail", "penal", "immobilier", "commercial", "autre") and optional `problemSubType` (string)
- `stepDescriptionSchema`: z.object with `description` (string, min 20), `urgency` (enum: "normal", "urgent", "tres_urgent", default "normal"), optional `opposingParty`, optional `desiredOutcome`
- `stepDocumentsSchema`: z.object with `documents` array of objects containing `id` (string), `name` (string), `size` (number), `type` (string), `encryptedKey` (string), `s3Key` (string), `nonce` (string), default empty array
- `stepContactSchema`: z.object with `fullName` (string, min 2), optional `phone`, `preferredContact` (enum: "email", "telephone", "les_deux", default "email"), optional `availabilities`
- `intakeSchema`: merge of all 4 step schemas
- `IntakeFormData`: z.infer type of intakeSchema
- `STEP_SCHEMAS`: array `[stepProblemTypeSchema, stepDescriptionSchema, stepDocumentsSchema, stepContactSchema]` for step-level validation

Add to `packages/shared/src/index.ts`:
```typescript
export * from "./schemas/intake";
```

**Step 6: Create Drizzle intake schema**
Create `apps/web/src/lib/db/schema/intake.ts` following exact Drizzle conventions from auth.ts (text IDs with crypto.randomUUID(), relations alongside tables):
- `intakeSubmissions` table with columns: id (text PK, $defaultFn crypto.randomUUID), userId (text, references users.id, nullable), problemType (text, notNull), problemSubType (text), description (text, notNull), descriptionNonce (text), urgency (text enum ["normal","urgent","tres_urgent"], default "normal"), opposingParty (text), desiredOutcome (text), fullName (text, notNull), phone (text), preferredContact (text enum ["email","telephone","les_deux"], default "email"), availabilities (text), status (text enum ["draft","submitted","assigned","reviewed"], default "draft"), createdAt (timestamp mode date defaultNow notNull), updatedAt (timestamp mode date defaultNow notNull)

- `intakeSubmissionsRelations` and `intakeDocumentsRelations` using drizzle-orm `relations()`
