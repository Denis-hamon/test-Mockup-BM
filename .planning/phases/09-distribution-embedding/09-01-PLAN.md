    **Create three Next.js API routes (per D-06):**

    **1. /api/widget/route.ts (GET):**
    - Read widget.js from `../widget/dist/widget.js` (resolve path relative to project root).
    - Return with Content-Type application/javascript.
    - CORS headers: Access-Control-Allow-Origin: *.
    - Cache control: immutable if ?v= param present, else max-age=3600.
    - Handle OPTIONS for preflight.

    **2. /api/widget/template/[slug]/route.ts (GET + OPTIONS):**
    - Internally query DB for template by slug (reuse getTemplateBySlug logic but as direct DB query, not server action).
    - Return template JSON with CORS headers.
    - 404 if not found.

    **3. /api/widget/submit/route.ts (POST + OPTIONS):**
    - Accept { slug, responses }.
    - Create intake submission in DB (reuse existing intake submission logic).
    - Return 201 with { success: true, id }.
    - CORS headers on all responses.

    **4. Create shared corsHeaders() helper** used by all three widget API routes. Place in `apps/web/src/app/api/widget/cors.ts` or inline if simple enough.
