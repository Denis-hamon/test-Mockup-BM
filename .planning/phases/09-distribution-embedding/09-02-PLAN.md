    - On click: toggle modal. Enter/Space keyboard support.
    - Fade-in animation class "lc-fade-in" on mount.

    - Container div "lc-modal" with slide-up animation "lc-slide-up".
    - role="dialog", aria-modal="true", aria-labelledby="lc-modal-title".
    - Focus trap: on mount, focus first focusable element. Tab cycles within modal.
    - Contains: WidgetHeader + WidgetIntakeForm (or WidgetConfirmation if submitted) + WidgetFooter.
    - Mobile: add backdrop div "lc-backdrop" behind modal.

    **4. WidgetHeader.tsx:**
    - Lawyer firm name (from template data) as modal title, id="lc-modal-title".
    - Close button (X SVG inline), aria-label="Fermer", onClick close handler.

    **5. WidgetFooter.tsx (per UI-SPEC copywriting):**
    - "Propulse par LegalConnect" text.
    - Lock icon (inline SVG) + "Chiffre bout en bout" text.
    - className "lc-modal-footer".

    **6. WidgetIntakeForm.tsx (simplified DynamicStepper reimplementation):**
    - Receives template steps/questions from Widget parent.
    - State: currentStep (number), formData (Record<string, unknown>), errors (Record<string, string>), submitting (boolean).
    - Renders form fields from template questions: text -> input, textarea -> textarea, select -> native select, checkbox -> checkbox, date -> input type="date", phone -> input type="tel", email -> input type="email".

    - Custom inline error display (red border + error message below field).
    - Conditional visibility: check question.conditionalRules (same logic as isQuestionVisible from shared schema, but re-implemented inline without Zod dependency).
    - "Suivant" button validates current step fields before advancing (check validity via form.reportValidity or manual required check).
