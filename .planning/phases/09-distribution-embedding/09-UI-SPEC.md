# Phase 9 — UI Design Contract

> Visual and interaction contract for embeddable widget and hosted lawyer pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui CLI v4 |
| Preset | base-nova (neutral base) |
| Component library | Radix UI (via shadcn) — main app only. Widget uses lightweight custom components. |
| Icon library | Lucide (main app), inline SVGs in widget (no external icon dependency) |
| Font | Geist (main app), system-ui stack in widget (avoids font loading in external sites) |

### Widget-Specific Design System Note

The widget runs inside Shadow DOM on external sites. It MUST NOT import shadcn components, Tailwind, or load external fonts. All widget styles are self-contained CSS within the Shadow DOM. The widget uses a minimal subset of the app's design tokens, hardcoded as CSS custom properties in the widget bundle.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding in widget button |
| sm | 8px | Compact element spacing, widget modal padding inline |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, widget modal padding block, landing hero padding |
| xl | 32px | Layout gaps, landing page section gaps |
| 2xl | 48px | Landing page hero vertical padding |
| 3xl | 64px | Landing page top/bottom margin |

Exceptions: 44px minimum touch target for floating widget button on mobile. 56px floating button diameter on desktop for visibility on external sites.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form fields in widget, landing page descriptions, integration instructions |
| Label | 14px | 600 (semibold) | 1.4 | Widget header, snippet labels, section titles in integration page |
| Heading | 20px | 600 (semibold) | 1.2 | Landing page section headings, integration page title, widget modal title |
| Display | 28px | 600 (semibold) | 1.2 | Landing page hero heading (lawyer/firm name) |

Note: Only 2 weights -- 400 (regular) and 600 (semibold). Consistent with Phases 6-8.

### Widget Typography Override

Inside the widget Shadow DOM, font sizes are declared in rem relative to a 16px root reset on the shadow host. Font family: `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`. No external font loaded.

## Color

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) | Page background (landing, integration settings) |
| Secondary (30%) | --card / --muted | oklch(1 0 0) / oklch(0.97 0 0) | Snippet code block background, landing hero card, specialties cards |
| Primary | --primary | oklch(0.205 0 0) | CTA buttons, active states in integration page |
| Destructive | --destructive | oklch(0.577 0.245 27.325) | Not actively used in this phase (no destructive actions) |
| Trust green | --trust (custom) | hsl(142 71% 45%) | Encryption badge in widget footer and landing page |

### Widget Color System

The widget injects lawyer accent color from `data-color` attribute. All widget internal colors derive from this single accent plus neutral grays.

| Widget Token | CSS Variable | Default | Source |
|-------------|-------------|---------|--------|
| Accent | --lc-accent | #1a365d | data-color attribute on script tag |
| Accent foreground | --lc-accent-fg | #ffffff | Auto-computed: white if accent luminance < 0.5, dark otherwise |
| Surface | --lc-surface | #ffffff | Hardcoded |
| Surface foreground | --lc-fg | #1a1a1a | Hardcoded |
| Muted | --lc-muted | #f5f5f5 | Hardcoded |
| Border | --lc-border | #e5e5e5 | Hardcoded |

### Hosted Page Color Override

On `/cabinet-[slug]` pages, lawyer accent color applied via `--lawyer-accent` CSS variable (same as Phase 8 /intake/[slug]). Applied to: hero CTA button, section accents, stepper progress.

Accent reserved for: floating widget button background, widget CTA button, landing page primary CTA, stepper progress fill. Never for decorative elements.

## Interaction Contract

### Floating Widget Button

| Interaction | Behavior |
|-------------|----------|
| Button click | Opens widget modal with slide-up animation (300ms ease-out). Button remains visible behind modal. |
| Button hover | Scale 1.05 transition (150ms). Subtle shadow increase. |
| Close modal (X) | Slide-down animation (200ms). Modal disappears. Button remains. |
| Close modal (backdrop click) | Same as X close. Only on desktop. |
| Escape key | Closes modal. Focus returns to floating button. |
| Widget load | Button appears after script loads with fade-in (300ms delay, 200ms fade). No layout shift on host page. |

### Widget Intake Form

| Interaction | Behavior |
|-------------|----------|
| Step navigation | "Suivant" validates current step before advancing. "Precedent" always available. |
| Step indicator dots | Completed steps filled with accent. Current step has ring. Future steps outlined. Not clickable (linear flow in widget). |
| Form submit | "Soumettre ma demande" button. Loading spinner while submitting. On success: show confirmation message in modal ("Votre demande a ete envoyee. Nous vous contacterons bientot."), auto-close after 3 seconds. |
| Field validation | Native HTML5 validation + inline error messages. Red border on invalid fields. |
| Scroll | Modal content scrollable if form exceeds modal height. Sticky footer (co-branding). |

### Hosted Landing Page

| Interaction | Behavior |
|-------------|----------|
| CTA "Commencer ma demande" | Smooth scroll to intake section below (#intake anchor). |
| Specialty cards | Static, no click action. Informational only. |
| Intake form | Same behavior as Phase 8 /intake/[slug] -- validates per step, auto-saves to localStorage. |

### Integration Settings

| Interaction | Behavior |
|-------------|----------|
| Copy snippet | Clipboard API. Button text changes to "Copie !" for 2s. Toast "Code copie dans le presse-papier." |
| Copy hosted link | Clipboard API. Toast "Lien copie dans le presse-papier." |
| Widget preview | Static preview, not interactive. Shows floating button + modal appearance with current lawyer branding. |

## Loading & Error States

| View | Loading State | Error State |
|------|---------------|-------------|
| Widget script load | No visual until ready. Button fades in once bundle initialized. | Silent failure: no button shown. Console.warn only. |
| Widget modal open | Skeleton: header + 3 form field placeholders (200ms max before real content). | Inline message: "Impossible de charger le formulaire. Rechargez la page." |
| Widget form submit | Button shows spinner, disabled. Form fields disabled. | Toast-like message inside modal: "Erreur lors de l'envoi. Veuillez reessayer." Form re-enabled. |
| Landing page load | Full-page centered skeleton: hero placeholder + 3 specialty cards + stepper skeleton. | Full-page centered alert: "Ce cabinet n'existe pas ou n'est plus disponible." |
| Integration page | Snippet pre-filled from profile. If profile incomplete, alert: "Completez votre profil cabinet pour generer le code d'integration." | Alert with link to /settings/cabinet. |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Widget button | role="button", aria-label="Ouvrir le formulaire de contact", tabindex="0". Keyboard: Enter/Space opens modal. |
| Widget modal | role="dialog", aria-modal="true", aria-labelledby pointing to modal title. Focus trapped inside modal when open. |
| Widget close | aria-label="Fermer". Focus returns to floating button after close. |
| Widget form | Standard form accessibility: labels linked to inputs, aria-required, aria-invalid, aria-describedby for errors. |
| Floating button position | Does not overlap host page scroll or fixed elements. Respects prefers-reduced-motion: no animations. |
| Landing page | Semantic HTML: header, main, section, footer. h1 for firm name, h2 for section headings. |
| Landing page image | Lawyer photo: alt="{Lawyer name}, avocat". If no photo: decorative placeholder hidden from screen readers. |
| Color contrast | Widget accent auto-contrast check: if accent luminance < 0.5, text is white; otherwise dark. Minimum 4.5:1 ratio enforced. |
