## Interaction Contracts

### I-01: AI Chat Zone Transition (D-06)

**Trigger:** User clicks "Suivant" on steps 0, 1, or 2 (not step 3, the last step).

**Behavior:**
1. Step validates via react-hook-form `trigger()`.
2. If valid, `phase` state changes from `"step"` to `"chatting"`.
3. AIChatZone fades in (200ms ease-out opacity transition).
4. AI streams 1-3 follow-up questions via `useChat` connected to `/api/chat/intake`.
5. Streaming indicator: `Loader2` spinning icon (16px, `muted-foreground` color).
6. User types response in text input and presses Enter or clicks "Envoyer".
7. User can click "Passer" at any time to skip.
8. After exchange or skip, user clicks "Continuer" to advance to next step.
9. `phase` resets to `"step"`, `currentStep` increments.

**Skip behavior:** "Passer" button is always visible. Clicking it persists the exchange as `skipped: true` in `ai_follow_ups` table. The AI chat zone closes and "Continuer" becomes the only action.

**Not shown after step 3:** The last step (Contact) goes directly to submission.

### I-02: Chat Bubble Rendering

**AI messages (assistant role):**
- Align left
- Background: `bg-background` with `border` (1px solid border color)
- Border radius: `rounded-lg` (var(--radius))
- Padding: `px-3 py-2`
- Text: 14px / 400 / foreground color
- Max width: 85% of chat zone width

**User messages (user role):**
- Align right
- Background: `bg-primary` with `text-primary-foreground`
- Border radius: `rounded-lg`
- Padding: `px-3 py-2`
- Max width: 85% of chat zone width

**Streaming state:**
- Partial text renders incrementally (character-by-character via AI SDK streaming)
- Loader2 icon (16px, animate-spin) shown below last message while `status === "streaming"`

### I-03: Extraction Card (D-04)

**Trigger:** File upload completes successfully, BullMQ job queued.

**States:**

| State | Visual |
|-------|--------|
| pending | Skeleton loader (3 rows, height 16px each, spaced 8px) below file preview |
| processing | Skeleton loader with pulsing animation + Badge "En cours d'analyse..." (variant outline) |
| done | Card with extracted fields: dates, parties, montants, clauses cles, type, resume |
| failed | Alert variant="destructive": "L'analyse automatique a echoue. Vous pouvez ajouter les informations manuellement." |

**Card layout (done state):**
```
+------------------------------------------+
| [DocumentType badge]     [Edit pencil]   |
|------------------------------------------|
| DATES         | 2024-03-15, 2024-06-01   |
| PARTIES       | Jean Dupont, SAS Acme    |
| MONTANTS      | 15 000 EUR               |
| CLAUSES CLES  | Article 3.2 - Resilia... |
|------------------------------------------|
| Resume: Contrat de travail entre...      |
+------------------------------------------+
```

- Container: `Card` component with `p-4`
- Field labels: 12px / 500 / muted-foreground / uppercase / tracking-wide
- Field values: 14px / 400 / foreground
- Each field is inline-editable: click to edit, shows Input component, blur to save
- Edit icon (Pencil, 14px) in top-right corner toggles all fields to edit mode
- `userEdited` flag set to true when any field is modified
- Vertical spacing between field rows: 8px

### I-04: Sensitive Case Alert (D-08)

**Trigger:** AI detects sensitive keywords in user text (violence, danger, suicide, menace). Detection happens within the LLM system prompt, not client-side.

**Visual:** The AI response includes a support message block. Render it as a distinct styled block within the chat:

- Background: warm amber tint `bg-amber-50 border-amber-200` (light mode) / `bg-amber-950/20 border-amber-800` (dark mode)
- Left border: 3px solid amber-500
- Icon: `Heart` (lucide) in amber-600, 16px
- Text: 14px / 400 / 1.5
- Emergency numbers displayed as a list: `3114`, `17`, `119` with labels
- No blocking of form — user continues normally after reading

**Detection method:** The LLM system prompt (INTAKE_FOLLOWUP_OVERLAY) instructs the AI to include the support message when sensitive markers are detected. The frontend renders it as styled content within the normal message flow. No separate client-side keyword detection.

### I-05: Extraction Polling

**Mechanism:** Poll tRPC query every 3 seconds while any extraction has status `pending` or `processing`. Stop polling when all extractions are `done` or `failed`.

**Transition animation:** When extraction completes (pending -> done), skeleton morphs to card with a 300ms fade-in transition.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Chat zone role | `role="log"` with `aria-live="polite"` for new messages |
| AI streaming | `aria-busy="true"` on chat container while streaming |
| Skip button | Keyboard accessible, visible focus ring, `aria-label="Passer les questions complementaires"` |
| Extraction card | `role="region"` with `aria-label="Informations extraites de [filename]"` |
| Editable fields | `aria-label` on each editable field, Enter to confirm, Escape to cancel |
| Sensitive alert | `role="alert"` for emergency number block |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 minimum). Amber alert text on amber bg verified at 7:1+ |

## Animation Contract

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Chat zone appear | Opacity 0 -> 1 | 200ms | ease-out |
| Chat zone disappear | Opacity 1 -> 0 | 150ms | ease-in |

| Extraction card appear | Opacity 0 -> 1 | 300ms | ease-out |
| Streaming spinner | Tailwind `animate-spin` on Loader2 | continuous | linear |

Use CSS transitions (Tailwind classes) for all animations. Framer Motion is available but not required for Phase 4 — CSS transitions are sufficient for these simple opacity/transform animations.
