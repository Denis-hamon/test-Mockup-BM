**4. Create Docling service** (`apps/web/src/server/services/docling.service.ts`):

Export `extractWithDocling(fileBuffer: Buffer, fileName: string)` that:
- POSTs to `${DOCLING_URL}/v1/convert/file` as `multipart/form-data` (NOT the URL-based endpoint — because files are SSE-C encrypted, we pass decrypted buffer per Pitfall 1)
- `DOCLING_URL` reads from `process.env.DOCLING_URL || "http://localhost:5001"`
- Constructs FormData with the buffer as a file blob
- Parses the response to extract text, tables, metadata
- Returns structured `DoclingResult`:
  ```typescript
  interface DoclingResult {
    text: string;
    tables: Array<{ headers: string[]; rows: string[][] }>;
    metadata: Record<string, unknown>;
  }
  ```
- On error, throws with status code for retry logic
- On Docling unavailable (ECONNREFUSED), throw specific error for fallback detection

**5. Create Vision extraction service** (`apps/web/src/server/services/extraction.service.ts`):

Export `extractWithVision(imageBuffer: Buffer, mimeType: string)` that:
- Uses `generateText` from `"ai"` (NOT streaming — extraction is batch)
- Uses `getModel({ provider: "anthropic" })` from `@legalconnect/ai`
- Sends multimodal message with image content part:
  ```typescript
  messages: [{
    role: "user",
    content: [
      { type: "image", image: imageBuffer, mimeType },
      { type: "text", text: EXTRACTION_PROMPT }
    ]
  }]
  ```
- `EXTRACTION_PROMPT` asks for JSON output with fields: `dates` (ISO format array), `parties` (name array), `montants` (array of `{value, currency}`), `clauses_cles` (string array), `type_document` (string), `resume` (2-3 sentences)
- Uses `maxOutputTokens: 1000`, `temperature: 0.1` (low for extraction accuracy)
- Parses `result.text` as JSON, returns typed `ExtractionResult`:
  ```typescript
  export interface ExtractionResult {
    dates: string[];
    parties: string[];
    amounts: Array<{ value: string; currency: string }>;
    keyClauses: string[];
    documentType: string;
    summary: string;
  }
  ```
- On JSON parse failure, return partial result with what's available

Also export `parseDoclingToExtractionResult(doclingResult: DoclingResult): ExtractionResult` — a function that uses `generateText` to extract structured data from Docling's raw text output. Same extraction prompt but with text input instead of image.

**6. Create BullMQ worker** (`apps/web/src/server/workers/extraction.worker.ts`):

Define `ExtractionJobData` interface:
```typescript
interface ExtractionJobData {
  documentId: string;
  s3Key: string;
  mimeType: string;
  fileName: string;
  encryptedKey: string; // base64 sealed box — needed to derive SSE-C key for S3 download
  nonce: string;
}
```

Create BullMQ `Queue` and `Worker` for queue name `"document-extraction"`:
- Connection: `{ host: process.env.VALKEY_HOST || "localhost", port: parseInt(process.env.VALKEY_PORT || "6379") }`
- Worker concurrency: 3
- Worker logic:
  1. Update extraction_results status to `"processing"`
  2. Download file from S3 using SSE-C key (derive from encryptedKey via `crypto_generichash(32, decryptedKey)` — same derivation as in use-file-encryption.ts). NOTE: The worker needs the raw file key, not the sealed box. For v1, pass the SSE-C key hash directly in job data since the worker runs server-side.
  3. Route by MIME type:
     - `application/pdf` -> `extractWithDocling(buffer, fileName)` then `parseDoclingToExtractionResult()`
     - `image/*` -> `extractWithVision(buffer, mimeType)`
     - Other types -> return `{ status: "skipped", reason: "unsupported_type" }`
  4. If Docling fails (ECONNREFUSED or 5xx), fallback to `extractWithVision()` for PDFs too (per Claude's discretion: Docling error fallback)
  5. Save result to `extraction_results` table with status `"done"`, or `"failed"` with error message
  6. Return `{ status: "done", documentId }`

Export the queue as `extractionQueue` for use by the trigger action.
