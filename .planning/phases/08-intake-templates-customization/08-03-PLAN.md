  <action>
    **1. Create useDynamicIntakeForm hook** at `apps/web/src/hooks/use-dynamic-intake-form.ts`:

    Adapted from existing use-intake-form.ts pattern. Props: { template: IntakeTemplate, slug: string }.

    - Compute total steps: 1 (Contact) + template.steps.length + 1 (Documents) = N steps.
    - Use react-hook-form with dynamic Zod resolver. The resolver changes per step:
      - Step 0 (Contact): use stepContactSchema from @legalconnect/shared
      - Steps 1..N-2 (specialty): use buildStepSchema(template.steps[stepIndex-1].questions) from shared package. Only validate VISIBLE questions (filter using isQuestionVisible before building schema).
      - Step N-1 (Documents): use stepDocumentsSchema from @legalconnect/shared
    - Register ALL fields across ALL steps at form initialization (per research Pitfall 3 — prevent data loss on step navigation). Default values: empty strings for text/textarea, undefined for select/date, false for checkbox, undefined for number.
    - localStorage auto-save: key = `lc-intake-draft-${slug}`, save every 10s per UI spec. On mount, restore from localStorage if draft exists.
    - Export: { form, currentStep, totalSteps, stepLabels, validateStep, nextStep, prevStep, hasDraft, clearDraft }.
    - validateStep: trigger validation only for current step's fields. For specialty steps, only validate visible questions.

    **2. Create CobrandingFooter** at `apps/web/src/components/intake/cobranding-footer.tsx`:

    Simple component. No props needed.
    - Renders: "Propulse par LegalConnect" with LegalConnect text in semibold (14px per UI spec). Lock icon + "Chiffre bout en bout" trust badge.
    - Layout: flex justify-between, muted-foreground text.
    - LegalConnect text is NOT a link (co-branding, not advertising per D-11).

    **3. Create DynamicStepper** at `apps/web/src/components/intake/dynamic-stepper.tsx`:

    "use client". Props: { template: IntakeTemplate, branding: BrandingConfig, templateId: string }.

    This replaces the hardcoded IntakeStepper for template-driven intakes. Structure mirrors existing intake-stepper.tsx but reads from template.

    - Uses useDynamicIntakeForm({ template, slug: branding.slug }).
    - Stepper header: horizontal on desktop (flex), vertical on mobile. Steps: "Contact" + template step labels + "Documents". Active step highlighted with accent color. Completed steps show CheckCircle2. Completed steps clickable to go back (per UI spec).
    - Step content rendering:
      - Step 0: render StepContact (reuse existing component). Need to adapt form types — StepContact expects UseFormReturn<IntakeFormData> but our form is Record<string, unknown>. Create a thin wrapper that passes the right fields.
      - Steps 1..N-2: render DynamicStep for template.steps[currentStep - 1] with form and allQuestions (flatten all template questions for conditional rule evaluation).
      - Step N-1: render StepDocuments (reuse existing, same type adaptation as Contact).
    - Navigation:
      - "Precedent" button: always available except step 0. No validation on back (per UI spec).
      - "Suivant" button: validates current step before advancing. If invalid, show inline errors.
      - Final step: "Soumettre ma demande" button (per UI spec copywriting). Calls submitIntake with form values + templateId. On success: clearDraft, show success Alert. On error: toast error, re-enable form.
    - Conditional field animation: 200ms fade per UI spec (use CSS transition opacity).
    - Submit flow per D-03: submitIntake receives templateId, server creates snapshot automatically (handled in Plan 01's modified intake.actions.ts).
  </action>
